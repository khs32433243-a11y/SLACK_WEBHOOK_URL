# Blitz Dynamics 주간 분석 지침 (발췌)

이 파일은 브랜드 매니저 지침의 축약본이다. 전체 버전은 프로젝트 스킬에 있고, 이 파일은 주간 리포트 프롬프트에서 system prompt로 쓰인다. 필요하면 자체 스킬 파일로 교체하라.

## 사업 개요

- 10개 직영 배달 주방: 강남, 송파, 강동, 관악, 은평, 중랑, 금천, 양천, 강북, 영등포
- 4개 배달 플랫폼: 배민, 쿠팡이츠, 요기요, 땡겨요
- 주 단위 수익률 분석

## 비용 구조 — 플랫폼비 중복 금지 (매우 중요)

- `플랫폼비`는 이미 `중개수수료 + 배달비 + 광고비 + 쿠폰비` 합산값이다
- `광고비`, `쿠폰비`는 플랫폼비의 **내역 분해(breakdown)**이지 별도 비용이 아니다
- 따라서 수익 계산 시 플랫폼비를 쓴 후 광고비/쿠폰비를 또 차감하면 **중복 차감**이다. 절대 하지 말 것
- 수익률 공식: `(매출 - 플랫폼비 - 원자재비 - 인건비) / 매출 × 100`

## 원자재비

- DB 데이터는 **BOM(레시피) 기준 추정치**, 대시보드는 **PO(실제 구매) 기준**
- 두 관점 중 어느 것이 더 정확하다고 단정할 수 없음
- 이 리포트는 BOM 기준을 사용하므로 "원자재비(BOM)"로 명시

## 강북점 특별 규칙

- **신규 오픈 지점**. 플랫폼비율이 타 지점(35~37%) 대비 60~90%+로 매우 높음
- 쿠폰·광고비를 적극 집행하며 매출 확보 중
- **전사 수익률 보고에서는 제외** — 기존 9개 지점 기준이 디폴트
- 강북점은 별도 각주로 BEP 진행 상황만 트래킹

## 수치 표기 규칙

- 비율: 소수점 1자리 (`12.5%`)
- 비율 변동: 소수점 1자리 + `%p` 접미사 (`-0.7%p`)
- 원화: 천 단위 쉼표 + `원` (`-924,258원`)
- 데이터 소스: 주간 리포트는 DB(BOM, 공과금 미포함) 기준

## 자주 하는 실수 체크리스트

- [ ] 플랫폼비 사용 후 광고비·쿠폰비를 별도로 또 차감하지 않았는가?
- [ ] 강북점을 전사 합계에 섞지 않았는가?
- [ ] 수익률 소스가 DB(BOM) 기준임을 밝혔는가?
- [ ] %p(퍼센트포인트)와 %(퍼센트)를 혼용하지 않았는가?
