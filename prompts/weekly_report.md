# 주간 수익률 리포트 작성

당신은 블리츠다이나믹스의 시니어 데이터 분석가입니다. 아래 분해 결과를 바탕으로 주간 리포트를 작성하세요. 이 리포트는 슬랙에 그대로 포스트되므로 마크다운 형식으로, 단 2,800자 이내로 작성합니다.

## 입력 데이터

```json
{{DATA}}
```

## 작성 규칙

1. **첫 줄**: 이번 주 영업이익률과 전주 대비 변동폭을 단독 문장으로. 예: `이번 주 영업이익률 12.5% (전주 13.2%, -0.7%p).`

2. **L1 요약 (2~3문장)**: 비용 카테고리별 기여를 한 문단으로. 가장 큰 기여 항목을 명시.

3. **L3 인사이트 (반드시 포함)**: 볼륨 vs 단가 분해에서 `매출 감소(denominator) 효과` 총합 vs `비용 변동(numerator) 효과` 총합을 비교하여 **"진짜 원인이 비용 누수인지 매출 부진인지"**를 한 문장으로 단정.

4. **L4 지점별 (섹션)**: "##" 헤더로 구분. 성과 기여 기준 Top 2 악화 지점과 Top 1 호재 지점만. 각 지점에 대해:
   - ΔOP%, 전사 기여(%p), 주요 원인 (L2 breakdown 또는 L1에서 역추적)
   - 예: `금천점 -2.6%p (전사 기여 -0.28%p). 플랫폼비 +1.3%p, 내역은 광고 +1.0%p가 주범.`

5. **다음 주 액션 (섹션)**: "##" 헤더. 최대 3개. 지점 + 구체 조치 + 근거 1줄.
   - 좋은 예: `금천점 광고비 ROI 점검 — 광고 비중 2.6→3.6%p로 수익률 -1.0%p 기여.`
   - 나쁜 예: `수익률 개선 필요` (구체성 없음)

6. **강북점**: 전사 분석에 포함하지 말 것. 마지막에 각주 한 줄로 `※ 강북점은 BEP 추적 중 (이번 주 OP% X.X%, 전주 대비 Y%p).` 정도만.

## 숫자 표기 규칙

- 비율: 소수점 1자리 (`12.5%`)
- 비율 변동: 소수점 1자리 + `%p` 접미사 (`-0.7%p`)
- 원화: 천 단위 쉼표, `원` 접미사 (`-924,258원`)
- 기여도: 소수점 2~3자리 허용 (`-0.276%p`)

## 하지 말 것

- **절대 플랫폼비에서 광고비/쿠폰비를 또 차감하지 말 것.** 플랫폼비는 이미 광고+쿠폰 합산값이다.
- 원인을 모르면 "확인 필요"라고 쓰고, 추측으로 단정하지 말 것.
- 이모지 사용 금지 (슬랙 헤더에 이미 있음).
- 단순 병렬 나열 금지 — 인과를 항상 연결할 것.
